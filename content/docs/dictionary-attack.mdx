# Dictionary Attack Simulator

A dictionary attack tries likely passwords one by one. For each candidate, the attacker hashes the candidate with the same algorithm used by the target and compares the result with the target hash.

CryptoViz includes this simulator for education only. It demonstrates the attack pattern with local demo hashes and a capped candidate list.

## What the simulator shows

1. A password is converted into a demo hash.
2. A dictionary list is loaded.
3. Each candidate is hashed.
4. The computed hash is compared with the target hash.
5. The simulator stops if a match is found.

## Why common passwords are risky

Passwords such as `password`, `123456`, `qwerty`, `admin`, and `password123` are dangerous because they appear very early in common dictionaries. If a match appears quickly, the simulator marks the result as higher risk.

## Defenses

- Use long, unique passwords.
- Use a password manager.
- Enable multi-factor authentication.
- Rate-limit login attempts.
- Store passwords with slow password hashing such as bcrypt, scrypt, or Argon2.
- Never store raw passwords.

## Ethical use

Use this feature only with the demo hashes generated by CryptoViz or with systems you own and have explicit permission to test.
